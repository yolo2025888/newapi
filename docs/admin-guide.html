1. 快速开始

如果你刚部署完，先做这 5 件事即可。

1）登录后台

进入 /console，使用管理员账号登录。

2）设置服务器地址

把默认的 http://localhost:3000 改成你的真实域名。

3）接入上游渠道

新增 OpenAI、Claude、Gemini 等渠道。

4）创建测试令牌

创建一个 token，验证模型调用是否正常。

5）查看日志

确认没有明显报错，请求能正常计费和记录。

6）修改默认密码

避免后台管理员账号被直接撞库。

2. 后台入口

后台入口一般为：

正式环境： https://你的域名/console
本地环境： http://localhost:3000/console
建议把你的正式后台入口整理给运营或管理员，例如：https://mabi.cn/console

3. 首次必做配置

01

登录后台

先使用管理员账号进入后台控制台。

入口链接示例：/console
02

修改服务器地址

把系统里的 ServerAddress 从默认值改成你的正式域名。

例如：https://mabi.cn

03

新增上游渠道

添加至少一个可用渠道，例如 OpenAI、Claude 或 Gemini。

04

创建测试 Token

创建一个用于测试的令牌，确认接口请求能正常返回。

05

检查日志和首页

确认首页显示的 Base URL、日志记录、渠道状态都正常。

4. 修改首页地址（把 localhost 改成你的域名）

首页显示的 http://localhost:3000 来自系统配置里的 ServerAddress，不是前端硬编码。

如果这里不改，会影响首页展示、支付回调、OAuth 回调、邮件跳转等地址生成。
1

进入后台设置

登录后台后，找到和支付/站点地址相关的设置页。

2

把服务器地址改成正式域名

示例：

  • 错误：http://localhost:3000
  • 正确：https://mabi.cn
3

刷新首页验证

保存后刷新首页，确认 Base URL 已显示为你的真实域名。

首页地址示例：/

5. 用户管理

常见操作

  • 创建用户
  • 禁用/启用用户
  • 修改额度
  • 查看消费情况

建议

  • 测试账号和正式账号分开
  • 高消耗用户重点关注日志
  • 不要多人共用管理员账号

6. 令牌管理

令牌用于用户调用 API。建议一个项目一个 token，方便排查。

你可以做什么

  • 创建令牌
  • 删除令牌
  • 限制模型范围
  • 控制额度

最佳实践

  • 一个客户一个 token
  • 泄露后立即停用
  • 不要长期使用管理员主 token

7. 渠道管理

渠道是上游模型配置中心。你可以接入多个服务商，并按模型、分组、权重进行分发。

1

新增渠道

填写渠道类型、密钥、模型列表、分组和权重。

2

测试渠道

先验证是否可用，再放到正式流量里。

3

按用途分组

例如 openai-main、claude-backup、gemini-test。

推荐做法：生产渠道和测试渠道分开，高成本模型单独配置倍率。

8. 额度与计费

系统支持按照模型、渠道、分组设置倍率和额度消耗。

用户额度

用于控制用户总可用余额。

模型倍率

不同模型可配置不同扣费倍率。

批量更新

适合大批量调整价格配置。

9. 日志排查

用户说“调用失败”时

  • 先看请求日志
  • 确认命中的模型和渠道
  • 看上游返回的错误信息

用户说“额度扣错了”时

  • 看 token 对应用户
  • 核对模型倍率
  • 核对账单日志

10. 安全建议

务必修改默认密码、默认地址、默认 secret。不要把密钥、数据库密码、管理员密码发到聊天或截图里。
  • 启用 HTTPS
  • 管理员密码使用强密码
  • 定期轮换上游 API Key
  • 备份数据库和 .env
  • 限制管理员数量

11. 常见问题

Q1：首页显示 localhost 怎么办？

去后台把 ServerAddress 改成你的正式域名。

Q2：用户说模型调用失败怎么办？

先看请求日志，再看渠道可用性和上游错误信息。

Q3：自动部署成功但页面没变化怎么办？

检查是不是 push 到 main，以及服务器容器有没有重建成功。

Q4：后台入口在哪里？

一般是：/console
